def plot_decision_boundary(clf, xlim, ylim, grid_resolution):
    """Display how clf classifies each point in the space specified by xlim and ylim.
    
    - clf is a classifier (already fit to data).
    - xlim and ylim are each 2-tuples of the form (low, high).
    - grid_resolution specifies the number of points into which the xlim is divided
      and the number into which the ylim interval is divided. The function plots
      grid_resolution * grid_resolution points."""

    # ... your code here ...
    x_cut = np.linspace(xlim[0], xlim[1], num=grid_resolution)
    y_cut = np.linspace(ylim[0], ylim[1], num=grid_resolution)
    x_cut_tile = np.tile(x_cut, grid_resolution)
    y_cut_repeat = np.repeat(y_cut, grid_resolution)
    stacked = np.stack((x_cut_tile, y_cut_repeat), axis=-1)
    df = pd.DataFrame(stacked, columns=clf.feature_names_in_)
    colors = {-1: 'pink', 1: 'lightskyblue'}
    y_pred = clf.predict(df)
    for y in {-1, 1}:
        plt.plot(df[y_pred == y].iloc[:, 0], df[y_pred == y].iloc[:, 1], color=colors[y], label=f'Class {y}')
        plt.legend()

3e. Why is feature scaling unnecessary for an ID3 decision tree? Answer in a markdown cell.¶

... your answer here in a Markdown cell ...¶

Decision trees make splits by the specific values of features, and their scale is not important.Thus, scaling of the features doesn't affect the tree's structure or decisions.

However, kNN relies on distance metrics, which can be easily changed by feature scaling.
